<wc-month-year-date>

A form-associated web component for selecting a month and year. Renders a localised month <select> and a year <input> that participate in HTML forms natively via the ElementInternals API. No framework dependencies.

Installation

<script src="https://unpkg.com/@annoyingmouse/wc-month-year-date/wc-month-year-date.js" defer></script>

Or as a module:

<script type="module" src="https://cdn.skypack.dev/@annoyingmouse/wc-month-year-date/wc-month-year-date.js"></script>

Attributes

Attribute Description
name Form field name submitted with the form
value Initial value in YYYY-MM or YYYY-MM-DD format; both fields start empty when omitted
default Set to now to pre-fill both fields with the current month and year when no value attribute is present
disabled Disables both controls; value is excluded from form submission
readonly Prevents changes; value is still submitted with the form
required Marks the field as required; triggers constraint validation on submit
label Visible label rendered inside the component; clicking it focuses the month control
autocomplete Autocomplete hint forwarded to the internal controls; cc-exp and bday are automatically split into their -month / -year tokens

Public API

const el = document.querySelector('wc-month-year-date')

// Read current value (YYYY-MM-01, or "" when incomplete)
console.log(el.value)

// Read form field name
console.log(el.name)

// Always "month-year-date"
console.log(el.type)

// The associated <form> element (null if not inside a form)
console.log(el.form)

Examples

Basic

Without a value attribute both fields start empty:

<wc-month-year-date name="expiry"></wc-month-year-date>

Default to current month and year

Add default="now" to pre-fill both fields with the current month and year when no value attribute is set: 

<wc-month-year-date name="expiry" default="now"></wc-month-year-date>

Pre-filled value

Pass value="YYYY-MM" to pre-select a specific month and year: 

<wc-month-year-date name="dob" value="1990-06"></wc-month-year-date>

Inside a form

The component participates in form submission natively. On submit, the data entry is YYYY-MM-01 (always the first of the month), or absent when either field is empty. 

<form>
  <wc-month-year-date name="expiry" value="2027-08"></wc-month-year-date>
  <button type="submit">Submit</button>
</form>

Reading the value via JavaScript

The value property always reflects the current selection as YYYY-MM-01, or an empty string when either field is incomplete.

Current value: (none)

const el = document.querySelector('#js-demo')
el.addEventListener('change', () => {
  console.log(el.value) // e.g. "2026-03-01"
})

Required field

Add required to trigger browser constraint validation on form submit. The component reports valueMissing when both fields are empty and badInput when only one field is filled. 

<wc-month-year-date name="expiry" required></wc-month-year-date>

Disabled

Add disabled to prevent interaction. The value is excluded from form submission, matching native form element behaviour. 

<wc-month-year-date name="expiry" value="2027-08" disabled></wc-month-year-date>

Read-only

Add readonly to prevent changes while still submitting the value with the form. 

<wc-month-year-date name="expiry" value="2027-08" readonly></wc-month-year-date>

Autocomplete

Use autocomplete="cc-exp" on a card expiry field or autocomplete="bday" on a date of birth field. The component automatically forwards cc-exp-month / cc-exp-year (or bday-month / bday-year) to the internal controls so password managers and autofill work correctly. 

<wc-month-year-date name="expiry" autocomplete="cc-exp"></wc-month-year-date>

Theming

The select and input are exposed as CSS ::part() targets named month and year: 

wc-month-year-date::part(month) {
  border-radius: 4px;
}

wc-month-year-date::part(year) {
  width: 6rem;
}

Accessibility

  • The component has role="group" so screen readers announce the month and year controls as a single composite field
  • A default aria-label="Month and year" labels the group; override it with your own aria-label or aria-labelledby
  • Each control has an aria-label so screen readers announce "Month" and "Year" individually
  • The year field uses type="text" with inputmode="numeric" — keyboards on mobile show digits while desktop screen readers are not burdened with spin-button announcements
  • After the user first interacts, aria-invalid is set on the relevant control(s) and an aria-live="polite" region announces the validation message without interrupting the user
  • The component is form-associated — screen readers announce it as part of the form, and it works correctly with <label>
  • disabled, readonly, and required propagate to the internal controls; required also sets aria-required on each control
  • autocomplete is forwarded to the internal controls; cc-exp and bday are split into their -month / -year tokens automatically (WCAG 1.3.5 — Identify Input Purpose)
  • Constraint validation uses internals.setValidity() and formResetCallback restores the initial value and clears error state when the form is reset
